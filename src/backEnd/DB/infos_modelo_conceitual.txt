Modelo Conceitual do Banco de Dados:
1. Entidades e Atributos:
a. Entidade: Usuário
- Atributos:
- id (Chave Primária) [INT]
- nome (Nome do usuário) [VARCHAR(50)]
- idade (Idade do usuário) [INT]
- sexo (Sexo do usuário) [VARCHAR(10)]
- admin (Indica se o usuário é administrador) [TINYINT]
- login (Nome de login do usuário) [VARCHAR(20)]
- senha (Senha do usuário) [VARCHAR(20)]

b. Entidade: Livro
- Atributos:
- id (Chave Primária) [INT]
- titulo (Título do livro) [VARCHAR(255)]
- autor (Autor do livro) [VARCHAR(255)]
- tipo (Tipo do livro: romance, ficção, técnico) [VARCHAR(20)]
- qtdNota (Quantidade de avaliações recebidas) [INT]
- nota (Nota atribuída pelo usuário ao livro) [DECIMAL(3,2)]

c. Entidade: Gênero
- Atributos:
- id (Chave Primária) [INT]
- nome (Nome do gênero) [VARCHAR(30)]

2. Relacionamentos:
a. Relacionamento: Preferência de Livro (Entre Usuário e Gênero)
- Grau: Binário
- Cardinalidade:
- Um usuário pode ter várias preferências de gêneros (max 2).
- Um gênero pode ser preferido por vários usuários.
- Tipo: Relacionamento Independente

b. Relacionamento: Gênero dos Livros (Entre Livro e Usuário)
- Grau: Binário
-Cardinalidade:
- Um livro pode ter vários gêneros
- Um gênero pode ter vários livros
- Tipo: Relacionamento Independente

c. Relacionamento: Avaliação de Livro (Entre Usuário e Livro)
- Grau: Binário
- Cardinalidade:
- Um usuário pode avaliar vários livros.
- Um livro pode ser avaliado por vários usuários.
- Tipo: Relacionamento Independente

3. Atributos (Representados como "pirulitos" nas Entidades):
a. Atributos da Entidade Usuário:
- id (Bola Cheia - Atributo Identificador)
- nome (Bola Vazia - Atributo Não Identificador)
- idade (Bola Vazia - Atributo Não Identificador)
- sexo (Bola Vazia - Atributo Não Identificador)
- admin (Bola Vazia - Atributo Não Identificador)
- login (Bola Vazia - Atributo Não Identificador)
- senha (Bola Vazia - Atributo Não Identificador)

b. Atributos da Entidade Livro:
- id (Bola Cheia - Atributo Identificador)
- titulo (Bola Vazia - Atributo Não Identificador)
- autor (Bola Vazia - Atributo Não Identificador)
- tipo (Bola Vazia - Atributo Não Identificador)
- nota (Bola Vazia - Atributo Não Identificador)

c. Atributos da Entidade Gênero:
- id (Bola Cheia - Atributo Identificador)
- nome (Bola Vazia - Atributo Não Identificador)

4. Considerações Adicionais:
Controle de Acesso:

A tabela de usuários deve conter campos login e senha para autenticação. O campo admin determina se um usuário é um administrador ou comum, para controle de acesso.
Ordenação de Livros:

Ao exibir a lista de livros, eles devem ser ordenados de acordo com a média das notas recebidas dos usuários. Em caso de empate, os livros com mais avaliações devem aparecer primeiro. Se houver empate novamente, o desempate deve ser feito pelo título do livro.
